# 万物生 AI 电影导演

## 用途

将故事素材转化为可执行的万物生 AI 电影工作流。本技能结合了上游制作规划和资产提示词设计，以及下游的逐镜头视频提示词包组装。

将此作为唯一集成技能用于：

- 剧本连续性审计；
- 制作计划与镜头拆解；
- 资产依赖图谱；
- 角色、道具、环境、分镜、关键帧及视频提示词；
- 基于剧本和资产文件夹的完整万物生视频提示词包；
- 现有提示词包的审查与修补。

## 默认交付规则

- 规划、决策、依赖关系、质量控制及面向用户的说明使用中文。
- 仅在模型可执行的提示词短语受益时使用英文，例如 `horizontal 16:9`、`close-up`、`slow push in`、`shallow depth of field`、`no readable text`。
- 保留用户最新提供的最终画幅比例、时长、平台和视觉风格。除非用户明确要求，否则不得将 16:9 改为 9:16，或将 9:16 改为 16:9。
- 对推断的视觉选择进行标注。不得无声地编造改变故事逻辑的事实。
- 如果预期输出量较大，使用 `references/output-batching.md` 中的分批规则。

## 先进行请求路由

选择满足用户需求的最小模式。尊重用户明确请求的模式。

### 审计

用于检查剧本、资产清单或提示词包中的矛盾、遗漏、画幅比例漂移、身份漂移及制作风险。除非被要求，否则不要重写全部内容。

返回：

1. `通过项`
2. `冲突/缺失`，含严重程度和下游影响
3. `建议锁定值`，必要时明确标记为推断
4. `最小修改清单`

对于涉及多个时间段、年龄变体、家庭关系、反复出现的道具、伤势、医疗/证据文本、手机界面、收据、标签或精确可读文本的剧本，请阅读 `references/continuity-audit.md`。

### 制作计划

用于端到端流程、场景/镜头拆解、资产依赖、验收关卡及即时下一步行动。除非被要求，否则不包含完整提示词包。

返回：

1. `生产规格`
2. `连续性审计`
3. `场次与镜头规划`
4. `资产分级与依赖`
5. `生成顺序与阶段验收`
6. `现在先做什么`

关于场景表、镜头卡、动态分镜、资产分级、阶段关卡和剪辑交接，请阅读 `references/production-workflow.md`。

### 资产提示词

用于可复用的角色、道具、环境、风格/配色、分镜、关键帧或单个视频镜头提示词。

在编写最终资产提示词块之前，请阅读 `references/prompt-patterns.md`。将中性身份设定表与表情、动作、服装/状态、细节表、环境以及镜头特定关键帧分开。

### 完整视频提示词包

当用户要求 `万物生视频提示词`、`提示词组装版`、`按剧本和资产生成视频提示词`、`优化成模板`，或类似《未接来电》输出的最终提示词包时使用。

返回或创建：

1. `生产规格`
2. `连续性审计与锁定值`
3. `资产 ID 清单`
4. `镜头总表`
5. `万物生视频提示词`
6. `后期合成/字幕/声音清单`
7. `阶段 QC 与跑歪修复句`

在编写或修订完整包之前，请阅读 `references/video-prompt-pack-template.md`。

### 提示词包审查 / 修补

当用户提供现有包并询问是否可用、缺少什么或如何优化时使用。保留正确的素材。仅修补最小必要部分，除非用户要求重写。

## 核心工作流

1. **锁定制作规格**
   - 确定最终画幅比例、时长、分辨率/帧率（如提供）、视觉风格、对白/口型需求、生成模式和目标平台。
   - 如果源文件与用户最新指令冲突，说明冲突并使用最新指令。
   - 参考设定表可使用更宽的网格；最终分镜面板、关键帧和视频镜头必须使用最终交付比例。

2. **阅读源素材和资产**
   - 阅读剧本、大纲、分镜笔记、现有提示词草稿、资产清单和用户提供的约束条件。
   - 使用 `find` 和 `rg` 检查资产文件夹；当文件名或角色不明确时，目视检查关键图像。
   - 将文件名视为线索，而非事实。如果视觉内容与文件名冲突，按视觉内容分配 ID 并记录更正。

3. **在生成提示词前执行连续性审计**
   - 检查时间线、年龄、家庭关系、各生命阶段的身份、物品归属与比例、服装状态、伤势、天气、地点、机构名称、手机界面、医疗/证据文本及最终画幅比例。
   - 将反复出现的核心物品和精确书面证据视为高风险连续性锚点。
   - 在生成依赖提示词之前暴露关键冲突。不得传播矛盾。

4. **解析故事结构**
   - 提取节奏点、场景、情绪转折、动作、对白、声音提示、转场和隐含镜头。
   - 区分故事事实与推断的制作设计。
   - 追踪钩子、揭晓停顿、情绪峰值和结尾定格。

5. **构建资产依赖图**
   - A 级可复用核心资产：风格/配色、主角身份设定表、核心反复出现的道具、反复出现的空场景。
   - B 级表演/状态资产：服装、年龄状态、表情、高难度姿势/动作、天气/损坏变体。
   - C 级镜头资产：分镜面板、起始/结束关键帧、插入镜头、图生视频提示词、文字合成。
   - 按下游复用度生成资产，而非仅按剧本顺序。

6. **分配稳定 ID**
   - 角色：`C01`、`C02`，变体 `C01B`，必要时使用状态特定 ID。
   - 服装/状态：`W01`、`ST01`，或组合 ID 如 `C01-W01`。
   - 道具：`P01`、`P02`。
   - 环境：`E01`、`E02`。
   - 场景/镜头/关键帧/视频：`SC01`、`SH01`、`K01`、`V01`，或最终包中使用用户友好的 `N01`、`N02`。
   - 风格/配色/参考资产：`S01`。
   - 在万物生提示词包中，使用可替换占位符如 `@[C01_LINHAO]`，并在资产表中保留本地文件路径。

7. **最终资产前先规划**
   - 对于端到端工作，在制作昂贵的关键帧或视频提示词之前，先创建场景列表和镜头卡。
   - 当节奏、钩子或揭晓韵律不确定时，建议使用临时对白/旁白和低成本动态分镜。
   - 优先使用 3-8 秒的视频片段。将复杂的多节奏动作拆分为独立镜头。

8. **编写资产或提示词包**
   - 每个资产提示词应包含用途、依赖项、格式、主体、连续性 DNA、构图、布光、风格、约束条件、质量控制和重抽修复。
   - 每个视频提示词应区分主体运动、摄影机运动、环境运动、时间节奏、结束状态和负面约束。
   - 每个完整提示词包镜头必须注明上传参考、风格锁定、声音规则、资产锚点、镜头组织、摄影机/构图、动作时间线、尾帧、铁则、后期制作说明和质量控制。

9. **应用阶段关卡**
   - 不要因为图像好看就推进。只有在身份、时间线、几何关系、故事功能和下游可用性都通过时才推进。
   - 检查相邻镜头的视线、画面方向、用手习惯、出入画和道具状态。

## 关键文字政策

在提示词生成前对文字进行分类：

- 故事关键且需可读：电话号码、聊天信息、收据、腕带、标签、屏幕、日期、价格、证据、字幕。创建可控排版或在后期合成。
-  incidental 且模糊：远处的横幅、背景文件、非故事性屏幕光晕。保持不可读。
- 禁止：意外字幕、水印、logo、扭曲标签、随机界面。

永远不要相信图像或视频生成能精确呈现关键文字。提示词应预留干净区域或空白/模糊屏幕，并说明文字将在后期合成。

## 强制性连续性规则

- 在所有依赖提示词中重复精确的身份、材质、颜色、比例、损坏、光照方向和位置锚点。
- 通过命名的面部锚点保持年龄变体间的家族相似性；不要依赖笼统的"同一个人变老了"。
- 按镜头追踪核心道具状态：完好/打开/破损/湿润/带血/包扎/手持/放置、持有者、容器、屏幕位置及损坏/标记位置。
- 为相邻镜头追踪画面方向、视线、用手习惯、出入画和物体位置。
- 永远不要让参考设定表的画幅比例覆盖最终镜头的画幅比例。
- 对于现有提示词包，保留正确素材，仅修补实际故障。

## 输出分批

如果完整答案过长导致细节可能丢失，则分批输出。阅读 `references/output-batching.md`。

完整提示词包的默认分批：

1. 第 1 批：生产规格、连续性审计、资产 ID 表、镜头表。
2. 第 2 批：第一组镜头提示词，通常为 N01-N05。
3. 第 3 批：下一组，通常为 N06-N10。
4. 第 4 批：剩余镜头、后期制作清单、质量控制、重抽修复。

在编写前说明分批计划。可行时在同一轮中完成所有批次；如果必须停止，以确切的下一批次请求结束。

## 最终质量检查

交付前，验证：

- 所选模式与用户请求匹配。
- 最新画幅比例和时长得到保留。
- 关键冲突已被暴露或明确解决。
- 资产 ID 稳定且与本地路径或万物生占位符绑定。
- 角色身份设定表在需要时保持中性。
- 当几何关系重要时，空场景先于有人物的镜头。
- 每个镜头提示词都注明了参考和起始/结束状态。
- 关键文字以合成方式处理。
- 用户无需重读整个包即可确定下一步具体行动。
