# 万物生大型交付物输出分批

当请求的交付物过大，单次响应可能丢失镜头级细节、遗漏资产或压缩质量控制时，使用分批输出。

## 何时分批

当满足以下任一条件时，默认分批：

- 超过 8 个视频镜头；
- 超过 4 个可复用角色或 8 个可复用资产；
- 用户要求完整提示词包加资产提示词；
- 输出同时包含制作计划和最终可执行提示词；
- 存在多条时间线、年龄变体、医疗/证据文字、手机界面、收据、标签或其他精确文字合成；
- 预期响应过长，无法保持每个镜头的摄影机、动作、声音、尾帧和质量控制的具体性。

小型任务不需要分批，例如单个资产、单个镜头或简短审查。

## 分批计划模式

在编写大型输出之前，简要说明：

```text
我会分 {N} 批输出，避免镜头细节被压缩：
1. 生产规格 + 连续性审计 + 资产ID + 镜头总表
2. N01-N05 视频提示词
3. N06-N10 视频提示词
4. N11-N16 + 后期清单 + QC/跑歪修复
```

根据项目长度调整镜头范围。

## 分批内容规则

- 第 1 批必须锁定决策。后续批次不得无声地修改画幅比例、资产 ID、时间线或文字政策。
- 仅重复当前镜头组所需的资产锚点；除非必要，不要每批都重新打印完整资产表。
- 跨批次保持镜头 ID 稳定。
- 每批以清晰标记结束，例如：`下一批：N06-N10。`
- 如果可以自动继续，则无需等待用户确认即可继续。
- 如果响应限制或轮次中断导致无法继续，告知用户确切的下一批次请求。

## 每个镜头的最低细节

不要将镜头压缩到以下字段以下：

- 上传参考 ID；
- 风格锁定；
- 声音规则；
- 当前状态的资产锚点；
- 镜头组织；
- 摄影机与构图；
- 时间节奏；
- 尾帧；
- 铁则与负面约束；
- 后期文字/音频说明；
- 质量控制。
