# 连续性审计

在根据剧本构建依赖提示词或审查现有提示词包之前，使用此检查清单。

## 1. 故事事实台账

不加修饰地提取事实：

| 领域 | 检查项 |
|---|---|
| 时间线 | 确切年份、经过时间、闪回顺序、季节、一天中的时间 |
| 角色 | 姓名、各时期年龄、关系、职业、身体病史 |
| 地点 | 地名、室内/室外、行程逻辑、反复出现的空间特征 |
| 服装 | 时期、服装状态、因天气/工作/受伤导致的变化 |
| 道具 | 持有者、比例、年代、材质、损坏、内容物、每场状态 |
| 伤势 | 身体侧别、原因、可见状态、治疗、后续疤痕或限制 |
| 天气/光线 | 雨/晴状态、阳光方向、湿润连续性 |
| 证据文字 | 日期、价格、姓名、手机界面、收据、标签、笔迹 |
| 机构 | 医院/诊所/养老院/学校/公司的命名和功能 |

## 2. 算术与身份检查

- 根据年份计算年龄，而非独立接受描述性年龄。
- 检查所有时期的父子/母子年龄差。
- 验证"年轻/年老板本"描述是否保留身份锚点。
- 检查核心道具是否在物理上适合其所述持有者和动作。
- 检查同一物品是否被类似类别意外替换，例如成人布鞋变成儿童运动鞋。
- 检查购买日期、学年、技术、车辆、服装或建筑是否与时期匹配。

## 3. 核心道具状态台账

为每个故事节奏点创建一行：

| 场景/镜头 | 持有者 | 状态 | 损坏/标记位置 | 容器 | 由谁手持 | 屏幕位置 |
|---|---|---|---|---|---|---|

对反复出现的磨损、污渍、褶皱、铭文、断裂边缘和扣件使用精确锚点。如果故事区分两个相似物品，明确描述它们的对比。

## 4. 关键文字政策

对文字进行分类：

- **故事关键且需可读**：收据、信件、日期、价格、证据、标题。创建可控排版或合成图层。
- ** incidental 且模糊**：手机屏幕、远处横幅、文件。保持不可读，除非故事需要。
- **禁止**：意外字幕、水印、logo、扭曲标签。

永远不要依赖图像生成器生成精确的关键文字。单独生成纸张/屏幕/物体纹理，然后合成经过验证的字体。

## 5. 冲突严重程度

- **阻断级**：改变身份、时间线、归属、剧情逻辑、最终画幅比例或反复出现的核心物品。停止依赖提示词生成。
- **严重级**：缺少反复出现的角色、地点、服装状态或场景覆盖。在最终关键帧之前补充。
- **轻微级**：可逆向推断的视觉选择，如中性墙面颜色。提出并标注。

## 6. 审计输出

```text
ID: CONT-01
严重程度：阻断级 / 严重级 / 轻微级
来源事实：{事实 A} vs {事实 B}
下游影响：{受影响的资产/镜头}
建议锁定值：{一个一致的值}
备选方案：{仅在实质有用时}
状态：需用户决策 / 可安全推断修复 / 已解决
```

不要通过在后续提示词中选择一个值来掩盖冲突。记录一次解决方案，然后一致使用。
